# meteo
Решение тестового задания. К сожалению, из-за нехватки времени, так как основные рабочие часы посвящены текущей работе, был сделан вполне рабочий вариант сайта, но сырой, который можно дорабатывать. Потраченно человеко-часов: ~8. Основной код в файлах utils и views. Использованы: Python 3.10, Django, pandas, requests, bs4

## ТЗ
Создать сайт, где пользователь вводит название города и получается прогноз погоды на ближайшее время:
* Дополнительные задачи:
    - [ ] Тесты
    - [x] Запуск с помощью Docker
    - [ ] Автодополнение при вводе города
    - [x] При повторном посещении сайта предложение посмотреть погоду, в котором пользователь смотрел ранее
    - [ ] Сохранение истории поиска для каждого пользователя
    - [ ] API, показывающее статистику по поиску
## Работа сайта:
  * Весь сайт состоит из 1 страницы, на ней пользователь вводит название интересующего города
  * Если пользователь уже искал какой-то город, то данные об этом хранятся в coockie, доступно последние 5 запросов
  * Список из 5 последних уникальных городов, запросы по которым были отработаны, с предложением посмотреть погоду выводятся выше формы
  * Полученная информация о погоде выводится на 7 дней в табличном варианте, где каждая таблица это определенный день недели с почасовым прогнозом: температура, влажность, скорость ветра
  * Для обработки данных из API использован pandas
  * Координаты для запроса к API парсятся с википедии
## Улучшения, которые можно сделать и недоработки, которые нужно исправить:
  * Внешний вид сайта
  * Сохранение пользователей в бд для статистики и хранения информации о городах. Сейчас города, которые искал пользователь хранятся в куках
  * Не все города доступны, можно сделать парсинг координат более гибким или использовать другое API для получения координат по названию города
  * Можно позволить пользователю самому выбирать те параметры погоды, которые его интересуют, сейчас они статические
  * Обработку ошибок с поднятием сообщения об ошибке, его отображения пользователю
## Установка и запуск проекта
  Проверьте установлен ли Docker
```bash
docker version
```
Если Docker отсутствует, установите его. После проверки запутите.
Клонируйте рипозиторий на свое устройство или скачайте на свое устроийтво другим методом:
```bash
git clone https://github.com/Tenrai-chi/meteo.git
```
Перейдите в папку weather внутри проекта!
```bash
cd weather
```
Теперь вы можете создать образ. Вместо test_meteo можете указать свое собственное имя образа.
```bash
docker build --no-cache -t test_meteo .
```
Запустите контейнер на основе созданного образа. Порт 8080 установлен в Dockerfile, но вы можете его изменить
```bash
docker run -p 8080:8080  test_meteo 
```
Теперь проект работает на вашем [localhost](http://127.0.0.1:8080/)
